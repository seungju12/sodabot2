# 2026-04 변경 이력

## 2026-04-30

### 아키텍처/방향 정리

- 운영자용 웹 대시보드 구현 방향을 사전 검토함
- 대시보드 작업 전, DB 분리와 컨테이너 기반 운영 구조를 먼저 정리하는 방향으로 결정함
- 장기 방향을 PostgreSQL + API/대시보드 확장 가능한 구조로 정리함

### 인프라

- 로컬 및 EC2 운영 기준 DB를 SQLite에서 PostgreSQL로 전환함
- SQLite 런타임 지원 제거 후 PostgreSQL 단일 백엔드 기준으로 정리
- `DATABASE_URL` 기반 연결 방식으로 통일
- Docker Compose 기반 `postgres`, `bot` 운영 구조 정리
- Docker Compose plugin, Buildx plugin 자동 설치 로직 추가

### 데이터베이스 전환

- PostgreSQL 16 기준 Compose 구성을 추가함
- DB 계층을 PostgreSQL 전용 구현으로 정리함
- `voice_owner_service`의 SQLite 전용 UPSERT 문법을 PostgreSQL 호환 문법으로 교체함
- 기존 SQLite 데이터를 PostgreSQL로 이전하는 마이그레이션 경로를 정리함
- SQLite fallback, 관련 의존성, 잔여 코드 경로를 제거함

### 컨테이너/런타임

- bot 이미지를 빌드하는 Dockerfile과 `.dockerignore`를 추가함
- Compose에서 `postgres`, `bot` 서비스가 함께 올라오도록 구성함
- bot 컨테이너 내부 `DATABASE_URL`이 Compose 서비스명 `postgres`를 사용하도록 정리함
- Windows 로컬 개발 환경에서 async PostgreSQL 연결이 동작하도록 이벤트 루프 호환 처리를 반영함
