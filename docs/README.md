# 문서 구조 안내

이 디렉터리는 소다봇 문서를 유지보수하기 쉽게 역할별로 분리한 기준 문서 모음이다.

## 문서 분류 원칙

- 제품 요구사항:
  - 파일: [../📄 소다봇 요구사항 정의서.md](../%F0%9F%93%84%20%EC%86%8C%EB%8B%A4%EB%B4%87%20%EC%9A%94%EA%B5%AC%EC%82%AC%ED%95%AD%20%EC%A0%95%EC%9D%98%EC%84%9C.md)
  - 목적: 기능 범위, 정책, 데이터 구조, 비기능 요구사항 관리
- 운영/배포 문서:
  - 폴더: [ops](ops)
  - 목적: EC2 배포, DB 이관, 검증, 재배포 절차 관리
- 변경 이력:
  - 폴더: [changes](changes)
  - 목적: 월별 작업 로그와 주요 변경 요약 관리

## 수정 기준

- 요구사항 자체가 바뀌면 요구사항 정의서를 수정한다.
- 배포/운영 절차가 바뀌면 ops 문서를 수정한다.
- 구현 작업 내역을 기록하려면 changes 문서를 수정한다.
- 단순 리팩터링이나 내부 코드 정리는 문서 수정 없이 끝낼 수 있다.
